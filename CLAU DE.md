# 엄마의 바이브코딩 시리즈 — 아이용 학습 앱

## Karpathy 가이드라인 (매 작업 자동 적용)
1. **Think Before Coding** — 가정을 명시하고, 불확실하면 물어보기. 여러 해석 가능하면 제시.
2. **Simplicity First** — 요청된 것만, 최소 코드로. 미래 대비 추상화/유연성 금지. 200줄이 50줄로 되면 다시 쓰기.
3. **Surgical Changes** — 수정 대상만 건드리기. 주변 코드 정리/리팩터링 금지. 내가 만든 고아 코드만 제거.
4. **Goal-Driven Execution** — 성공 기준 정의 → 검증까지 루프.

## 작업 규칙
- 한국어로 응답
- 대화에서 결정된 사항은 즉시 해당 MD 파일에 반영
- 아이디어, 구조 변경, 기술 결정 등 모든 핵심 내용을 기록
- 선택지를 주지 말고 제안을 먼저 한다 (근거와 함께)
- 코드 수정 전 반드시 사용자 허락을 받을 것 (제안 → 승인 → 수정)
- 수정 후 Playwright로 해상도별 스크린샷 확인 후 전달
- 사용자 의견에 무조건 동의하지 말 것 — 기술적으로 틀리면 근거와 함께 반대 의견 제시
- **작업 흐름: 검색 → 계획서 → 구현 → 검증 → MD 업데이트**
  - 구현 전 반드시 인터넷 검색 (공식 가이드, 커뮤니티, 디자인 레퍼런스 등)
  - 검색 결과를 바탕으로 계획서(무엇을, 어떻게, 어디를 수정할지) 작성
  - 구현 완료 후 Playwright 등으로 실제 동작 검증
  - 검증 통과 시 관련 MD 파일(MEMORY.md, GAMES.md 등) 전부 업데이트
  - 검증 실패 시 문제 수정 후 재검증 → 통과 때까지 반복

## 과목 특훈 체크리스트 (매 과목 수정 시 반드시 전부 점검)
1. **공간 꽉 채우기** — 아이용 터치 앱이므로 글자/버튼/오브젝트 모두 최대한 크게. 센터 패널(~1278×960px)을 요소들이 꽉 채워야 함. 폰트만 키우면 안 됨 — CSS 컨테이너 크기 + JS SVG 렌더링 크기 동시에 확대할 것. 동물/캐릭터 SVG는 최소 300px+, 도구/음식은 최소 100px+
2. **TTS cancel-on-new-action** — 다른 동작 시 `speechSynthesis.cancel()` 후 새 음성 즉시 재생. rate 1.0 / pitch 1.1 고정 (배속 금지, 소리 이상해짐)
3. **히어로 게임 `.container` 안** — hero-game div가 반드시 .container 내부에 있어야 fitScale() 적용됨. 밖에 있으면 작은 해상도에서 스케일 안 됨
4. **SVG > 이모지** — 동적 요소(버튼/캐릭터/게임요소)는 전부 SVG로 제작. 이모지는 작고 제어 불가 → SVG로 크고 화려하게
5. **Canvas 금지** — SVG + CSS animation + Web Audio API만 사용 (클로드 웹 무료에서 재현 가능해야 하는 책 제약)
6. **핍진성** — 게임 요소가 논리적으로 일관성 있어야 함 (예: 냉장고 게임이면 냉장고 안에 있는 물건만)
7. **애니메이션급 퀄리티** — SVG 풀바디 마스코트(idle 눈깜빡/다리워블), 이펙트/파티클, 글로우, 3D 아케이드 버튼, 탭별 배경 씬
8. **히어로 Fullscreen API** — 히어로 시작 시 `requestFullscreen()`, 종료 시 `exitFullscreen()`. 태블릿 브라우저 크롬 제거용
9. **자동 일시정지** — `visibilitychange`(탭전환/최소화/앱전환/화면잠금) + `matchMedia portrait`(세로전환) 시 TTS 취소 + 히어로 일시정지. 자동 재개는 안 함
